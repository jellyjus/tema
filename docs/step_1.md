Шаг 1. Подключение к данным (Data Ingestion)
🎯 Цель

Получать актуальные данные по prediction markets (цены, рынки) на регулярной основе.
1. Источник данных

Используем API Polymarket.
Требуется:

    получать список активных рынков

    получать цены (probability)

    получать базовую информацию по рынку

2. Что нужно реализовать
   2.1 Запрос списка рынков

Система должна:

    отправлять запрос к API

    получать список рынков

Ожидаемый формат (упрощённо):

[
{
"id": "market_1",
"question": "Will X happen?",
"outcomes": ["Yes", "No"],
"prices": [0.65, 0.35],
"volume": 12345
}
]

2.2 Что извлекаем

Для каждого рынка:

market_id  
title (question)  
yes_price  
no_price  
volume  
timestamp (текущее время)

2.3 Преобразование данных

Важно:

    использовать только YES цену как probability

    привести всё к диапазону 0–1

probability = yes_price

3. Частота обновления

Система должна:

обновлять данные каждые 1–5 минут

(параметр должен быть настраиваемым)
4. Обработка ошибок

Обязательно предусмотреть:

    если API не отвечает → повтор запроса

    если данные пустые → пропустить итерацию

    логировать ошибки

5. Хранение (минимально)

На этом этапе достаточно:
Вариант 1 (предпочтительно):

простая БД (SQLite / Postgres)
Таблица:

markets:
- id
- title
- probability
- volume
- timestamp

Вариант 2 (если быстро):

JSON / CSV
6. Структура кода (рекомендация)

Разделить на модули:

data_fetcher.py  → запросы к API  
data_parser.py   → обработка данных  
storage.py       → сохранение

7. Выход шага (Definition of Done)

Шаг считается выполненным, если:

✅ система получает список рынков
✅ корректно извлекает цены
✅ сохраняет данные
✅ обновляется по таймеру
8. Простой псевдокод

while True:

    data = fetch_from_api()

    markets = parse(data)

    save(markets)

    sleep(UPDATE_INTERVAL)

9. Важно (ограничения MVP)

   не фильтровать рынки

   не анализировать

   не усложнять

👉 только сбор данных
