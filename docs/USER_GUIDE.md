# tema — Руководство пользователя

## Что это такое

tema — инструмент для поиска выгодных ставок на Polymarket. Он анализирует рынки, находит расхождения между рыночной ценой и «справедливой» вероятностью, и показывает, на что стоит ставить и сколько.

Вы не ставите реальные деньги через tema — это **бумажный трекер**. Вы определяете логические связи между событиями, а система рассчитывает, где рынок ошибается.

---

## Как это работает (простыми словами)

### Шаг 1. Данные подтягиваются автоматически

Сервис каждые 60 секунд забирает актуальные цены с Polymarket. Цена YES-контракта = вероятность события (от 0 до 1). Например, если цена YES = $0.65, рынок считает, что событие произойдёт с вероятностью 65%.

### Шаг 2. Вы задаёте связи между рынками

Связь — это ваше суждение «если X произойдёт, это повлияет на Y». Например:

- «Если FDA одобрит препарат, то цена акций компании вырастет» → **positive** связь с весом 0.8
- «Если начнётся рецессия, то рынок недвижимости упадёт» → **negative** связь с весом 0.6

Вы создаёте эти связи вручную на вкладке **Связи**.

### Шаг 3. Система считает ожидаемую вероятность

Для каждого рынка, у которого есть входящие связи, считается ожидаемая вероятность:

- **positive** связь: если источник растёт, цель тоже растёт
- **negative** связь: если источник растёт, цель падает

Все связи взвешиваются и нормализуются. Результат — число от 0 до 1, которое показывает, какой **должна быть** вероятность по вашей модели.

### Шаг 4. Находится edge (преимущество)

```
edge = ожидаемая вероятность − рыночная вероятность
```

Если ожидаемая 70%, а рынок даёт 55% → edge = +15% → **BUY YES** (рынок недооценивает).

Если ожидаемая 30%, а рынок даёт 55% → edge = −25% → **BUY NO** (рынок переоценивает).

### Шаг 5. Учитывается поведение толпы

Если цена резко скакнула с ростом объёма → система считает, что толпа давит на цену, и усиливает сигнал (confidence = 1.1). Иначе — нейтрально (1.0).

### Шаг 6. Рассчитывается размер ставки

```
ставка = банкролл × k × |adjusted_edge|
```

Ограничения:
- Минимум 1% от банкролла на ставку
- Максимум 5% от банкролла на ставку
- Рынки с экстремальной вероятностью (<10% или >90%) → ставка уменьшается вдвое
- Суммарно на все ставки — не больше 25% от банкролла

### Шаг 7. Открывается бумажная сделка

На каждый сигнал автоматически создаётся сделка с entry_price = текущей рыночной вероятности. Когда событие разрешается, вы закрываете сделку, указывая исход — и система считает P&L.

## Вкладки интерфейса

### Вкладка «Сигналы»

Главная вкладка. Показывает, куда система рекомендует ставить прямо сейчас.

| Столбец       | Что значит                                                                    |
|---------------|--------------------------------------------------------------------------------|
| **Рынок**     | Название события на Polymarket. Наведите курсор — увидите полный ID           |
| **Маркет.**   | Текущая рыночная вероятность (цена YES)                                       |
| **Ожид.**     | Вероятность, которую выдаёт ваша модель связей                                |
| **Edge**      | Разница между ожидаемой и рыночной вероятностью                               |
| **Adj.**      | Adjusted edge — edge с учётом поведения толпы (усилен, если толпа давит)        |
| **Напр.**     | BUY YES (рынок недооценивает) или BUY NO (рынок переоценивает)                  |
| **Ставка**    | Рекомендуемый размер ставки в $, рассчитанный sizer                           |
| **Сила**      | Слабый / Средний / Сильный — зависит от |adjusted_edge|                      |
| **Повед.**    | neutral (обычная ситуация) или crowd (цена скакнула с объёмом)                 |
| **Время**     | Когда сигнал сгенерирован                                                     |

**Цветовая кодировка:**
- Зелёный — положительный edge (преимущество в вашу сторону)
- Красный — отрицательный edge
- Сила: серый = слабый, жёлтый = средний, красный жирный = сильный

> Сигналы обновляются каждый fetch-цикл (каждые 60 сек по умолчанию). Старые удаляются, новые записываются.

---

### Вкладка «Связи»

Место, где вы определяете логику модели. Без связей сигналов не будет — системе не из чего считать ожидаемую вероятность.

**Форма добавления связи:**

| Поле           | Что вводить                                                                |
|----------------|----------------------------------------------------------------------------|
| **Источник**  | ID рынка-причины. Начните вводить — появится список с подсказками           |
| **Цель**       | ID рынка-следствия. Тоже с автоподсказками                                  |
| **Тип**        | `positive` — если источник растёт, цель тоже растёт. `negative` — наоборот |
| **Вес**        | От 0.01 до 1.0. Насколько сильно влияние. 1.0 = максимальное               |

**Таблица связей:**

| Столбец              | Что значит                                      |
|-----------------------|--------------------------------------------------|
| **ID**               | Внутренний номер связи                          |
| **Источник → Цель** | Названия рынков с направлением                   |
| **Тип**              | positive (зелёный) или negative (красный) тег    |
| **Вес**              | Число от 0.01 до 1.0                            |
| **✕**                | Удалить связь                                    |

**Примеры связей:**

| Источник                                    | Цель                                         | Тип       | Вес  | Почему                                    |
|---------------------------------------------|----------------------------------------------|-----------|------|-------------------------------------------|
| FDA approves Drug X                         | Company Y stock goes up                       | positive  | 0.8  | Одобрение прямо влияет на акции          |
| Recession declared                         | Housing market drops                          | negative  | 0.6  | Рецессия →问她 рынок жилья тормозит      |
| Fed lowers rates                            | S&P 500 goes up                              | positive  | 0.7  | Снижение ставок → рост акций             |
| War escalates in region X                  | Oil price rises                               | positive  | 0.5  | Геополитика → топливо                     |

> Чем больше связей для одного целевого рынка — тем точнее модель. 3–5 связей — хороший минимум.

### Вкладка «Сделки»

Трекинг P&L — сколько бы вы заработали или проиграли, если бы следовали сигналам.

**Метрики вверху:**

| Метрика    | Что значит                                                         |
|------------|---------------------------------------------------------------------|
| **PnL**    | Суммарная прибыль или убыток. Зелёный — в плюсе, красный — в минусе |
| **ROI**    | Возврат на вложенный капитал (PnL / общая сумма ставок)             |
| **Win Rate**| Процент выигранных сделок от общего числа                          |
| **Всего**  | Общее количество закрытых и открытых сделок                        |
| **Win/Loss**| Количество выигрышей и проигрышей                                   |

**Таблица сделок:**

| Столбец   | Что значит                                                            |
|-----------|------------------------------------------------------------------------|
| **Рынок** | Название события                                                       |
| **Напр.** | BUY YES (ставим на «да») или BUY NO (ставим на «нет»)                 |
| **Вход**  | Цена входа в центах. Это рыночная вероятность на момент открытия      |
| **Выход** | Цена закрытия в центах. «—» если сделка ещё открыта                   |
| **Ставка**| Размер ставки в $                                                     |
| **PnL**   | Прибыль (+) или убыток (−) в $. Только для закрытых сделок            |
| **Статус**| open (открыта) или closed (закрыта)                                    |
| **Открыт**| Дата открытия сделки                                                   |
|           | Кнопка «Закрыть» с полем для ввода цены                               |

**Как закрыть сделку:**

1. Найдите открытую сделку (статус = open)
2. В поле справа введите `exit_price` — цену закрытия:
   - `1.0` = событие точно произошло
   - `0.0` = событие точно НЕ произошло
   - `0.85` = событие произошло с вероятностью 85% (для частичных исходов)
3. Нажмите «Закрыть»

**Как считается P&L:**

Для BUY YES (ставим на то, что событие произойдёт):
- Событие произошло (exit = 1.0): **прибыль = ставка × (1 − цена входа)**
- Событие не произошло (exit = 0.0): **убыток = −ставка × цена входа**

Для BUY NO (ставим на то, что событие НЕ произойдёт):
- Событие не произошло (exit = 0.0): **прибыль = ставка × (1 − цена входа)**
- Событие произошло (exit = 1.0): **убыток = −ставка × цена входа**

**Пример:**
- BUY YES по цене 0.40, ставка $100
- Событие произошло → прибыль = $100 × (1 − 0.40) = **+$60**
- Событие НЕ произошло → убыток = −$100 × 0.40 = **−$40**

> Сделки создаются автоматически из сигналов. Дубли не создаются — на один рынок только одна открытая сделка.

---

### Вкладка «Рынки»

Справочник всех рынков, загруженных из Polymarket. Показывает ID, название и дату добавления. Используется в основном для того, чтобы найти ID рынка для создания связей.

## Настройки

Через переменные окружения (`.env` файл или env):

| Переменная               | По умолчанию  | Зачем менять                                                                       |
|--------------------------|---------------|-------------------------------------------------------------------------------------|
| `SIGNAL_THRESHOLD`      | `0.01`        | Минимальный edge для сигнала. Увеличьте до 0.10–0.15 для более редких, но сильных  |
| `BANKROLL`              | `1000`        | Ваш банкролл в $. Влияет на размер ставки                                           |
| `RISK_K`                | `0.5`         | Агрессивность ставок. 0.3 — консервативно, 0.7 — агрессивно                         |
| `MIN_VOLUME`            | `0`           | Минимальный объём рынка. Фильтрует неликвидные рынки                                |
| `PRICE_CHANGE_THRESHOLD`| `0.05`        | Порог для определения «толпы». Снизьте для большей чувствительности                |
| `FETCH_INTERVAL`        | `60`          | Секунды между обновлениями данных. Меньше — свежее, но больше нагрузки            |

## Рекомендации по использованию

1. **Начните с 3–5 связей** на один целевой рынок. Одна связь даёт шум, несколько — сигнал.
2. **Проверяйте силу сигнала** — сильные сигналы (adjusted edge > 25%) заслуживают внимания.
3. **Обращайте внимание на поведение** — `crowd` означает, что цена недавно скакнула. Это может быть возможностью или риском.
4. **Меняйте банкролл** под свой капитал. Больше банкролл — крупнее ставки (но пропорции те же).
5. **Уменьшите RISK_K**, если хотите консервативнее. Увеличьте — если готовы к риску.
6. **Закрывайте сделки** после разрешения события. Без этого метрики P&L не считаются.
