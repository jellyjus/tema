План разработки MVP (пошагово)
Шаг 1. Подключение к данным (Data Ingestion)

Что сделать:

    подключиться к API (например, Polymarket)

    получать список рынков

    получать текущие цены (probabilities)

Результат шага:
👉 есть сырые данные по рынкам в системе
Шаг 2. Сохранение данных

Что сделать:

    создать простую структуру хранения (БД или даже JSON)

    сохранять:

        id рынка

        название

        цену

        время

Результат:
👉 данные не теряются и доступны для расчётов
Шаг 3. Задание связей (relations)

Что сделать:

    вручную задать связи между рынками

    для каждой связи указать:

        источник (market A)

        цель (market B)

        тип (positive / negative)

        вес

Результат:
👉 система “понимает”, какие рынки связаны
Шаг 4. Реализация многофакторной модели

Что сделать:

    для каждого рынка собрать все влияющие факторы

    реализовать формулу:

expected = сумма (probability × weight)

Результат:
👉 система считает “ожидаемую вероятность”
Шаг 5. Расчёт расхождения (edge)

Что сделать:

    для каждого рынка считать:

edge = expected - market_price

Результат:
👉 появляется метрика mispricing
Шаг 6. Фильтрация сигналов

Что сделать:

    задать порог (например 10–15%)

    отсеивать слабые сигналы

Результат:
👉 остаются только значимые возможности
Шаг 7. Упрощённый поведенческий анализ

Что сделать:

    добавить простую логику:

        резкое движение + объём → “толпа”

    корректировать сигнал:

edge × коэффициент (например 1.1)

Результат:
👉 базовый учёт поведения рынка
Шаг 8. Генерация сигналов

Что сделать:

    формировать объект сигнала:

        рынок

        цена

        expected

        edge

        направление (BUY YES / NO)

Результат:
👉 готовые сигналы для принятия решений
Шаг 9. Вывод информации

Что сделать:

    реализовать простой вывод:

        таблица / консоль / файл

Результат:
👉 человек видит сигналы
Шаг 10. Цикличность работы

Что сделать:

    запускать процесс каждые N минут

    обновлять данные и пересчитывать

Результат:
👉 система работает постоянно
🔚 Итоговая логика

данные → связи → expected → edge → фильтр → сигнал
